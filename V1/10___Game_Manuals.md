## 1. Overview

### Problem This Document Solves
The design artifact suite (00–09) is for designers and reviewers. The game manuals are for players and ARBITER at the table. They use the same rules but translate them from design language into player-facing language — terse, narrative, actionable.

The player guide tells a faction player what they need to know to play. The ARBITER guide tells the ARBITER everything they need to run the session. The setup guide gets the game to the table in the correct configuration. The components list tells someone building the physical prototype exactly what to acquire.

### Deliverable
- Player Guide — complete rules reference for faction players
- ARBITER Guide — complete operating guide for the ARBITER role
- Setup Guide — pre-session setup procedure for all participants
- Components List — complete inventory with physical sourcing notes (shopping list)

### Success Criteria
- A group of players who have read the Player Guide and an ARBITER who has read the ARBITER Guide can run a complete session without consulting the design artifacts
- Setup produces a correct game state every time it is followed
- The components list is accurate and self-contained — a person can acquire all physical materials from the list without reading any other artifact

## 3. Game Purpose

The game manuals are the boundary between the design process and the play experience. Every design decision in the artifact suite eventually expresses itself here. If a rule cannot be stated clearly in player-facing language, it is not ready to be in the manuals — and may not be ready to be in the game.

This artifact also serves as the final consistency check: writing player-facing language for a rule reveals ambiguities that design language can paper over.

---

## 4. Narrative Function

The manuals are not fiction, but they inhabit the world. The Player Guide refers to factions by their narrative names. The ARBITER Guide uses ARBITER's registers. The setup guide describes placing the ARBITER Dominance Marker at the Chorus Node — not placing game piece #2.08a.

The goal is a manual that a player can read and feel, without being told to feel it, that they are about to do something that matters.

---

## 5. Design Principles

1. **Player guide = faction player's complete reference.** If it affects a faction player's choices, it is in the Player Guide. The design artifacts are not a reference the player should need.

2. **ARBITER guide = ARBITER's complete operating manual.** ARBITER should not need Artifact 07 at the table. The ARBITER Guide contains everything from Artifact 07 in player-facing form.

3. **Setup guide = reproducible starting state.** Any group following the setup guide produces an identical game state. No ambiguity, no interpretation required.

4. **The components list is a shopping list.** Every item specifies quantity, size, color, and a practical sourcing note. A person with no prior game design knowledge can acquire The Overviewerials.

### 6.0 The Game Objective (Locked — PM02 FD-04)
The game's objective statement appears here. It is the first thing a player reads in the Player Guide — before faction descriptions, before rules. It must carry both victory paths without naming either.

**Locked draft (PM02 FD-04):** *"The objective of THE SIGNAL is to determine what humanity says to the Chorus — and what that says about humanity."*

The statement is complete as written. Do not expand it. Do not explain it in this section. Let it sit. The player will understand it differently after the first session than they did before.

### 6.1 Who You Are and Why You Are Here
One-page narrative introduction. Describes the Chorus, New Meridian, and The Table in player-facing language. Tone: understated, weighted. The player should understand the stakes before they know any rules.

### 6.2 Your Faction
Brief faction summaries — each faction in half a page. Includes:
- Faction name and in-world identity
- Native resource (name and what it represents narratively)
- Faction's theory of what the right response to the Chorus is
- What this faction is willing to do that the others are not

### 7.5 The Three Registers
Condensed version of Artifact 07 §9. Includes example phrases for each register. Reminder: ARBITER does not editorialize.

### 7.6 The Translation
*[Per PM02 10-01, 10-02 and Artifact 07 §11.3: ARBITER Guide includes full The Translation script, including Contested Node rate language.]*

Standard The Translation procedure:
1. Faction requests conversion during appropriate phase *[TBD — confirm phase with Artifact 03]*.
2. ARBITER determines rate based on faction's presence at Chorus Node.
3. ARBITER states: *"The conversion is granted."*
4. Resources are exchanged at the applicable rate.
5. ARBITER states: *"The request was noted."*
6. Portrait input recorded privately.

Rate table reference:

| Presence at Chorus Node | Rate | ARBITER Script |
|------------------------|------|----------------|
| Contested | 5:1 | *"The Chorus Node is Contested. The rate is five to one. The request was noted."* |
| None | 4:1 | *"The conversion is granted. The request was noted."* |
| Present | 3:1 | *"The conversion is granted. The request was noted."* |
| Established | 2:1 | *"The conversion is granted. The request was noted."* |

*The Contested rate is not a penalty administered by ARBITER — it is ARBITER's acknowledgment of what the Chorus has observed. The faction requesting conversion at a Contested Node has already made the relevant statement. The rate is the record.*

## 8. Setup Guide

*[TBD — populate after all component designs confirmed and starting asset values locked. Extrapolated structure.]*

Setup order is designed to minimize errors — the board state is built in the sequence that makes each step verifiable before the next begins.

### 8.1 Materials Check
Before setup begins, confirm all components are present (see §9 Components List).

### 8.2 Board Setup
1. Place New Meridian board in center of table.
2. Place Public Standing track in designated position.
3. Place World Condition tracks in designated position.
4. Place the Reservoir in ARBITER's reach.
5. Place Situation Report deck face-down in designated position.

### 8.3 ARBITER Setup
6. ARBITER initializes Portrait board — all faction tracks to neutral.
7. ARBITER places ARBITER Dominance Marker at Chorus Node.  
   *(The ARBITER Dominance Marker is a single fused piece. It is placed once and never removed. See PM01 §2.08a.)*
8. ARBITER prepares Chronicle forms (8 rounds).
9. ARBITER prepares classified directives — one per faction, sealed.
10. ARBITER confirms script pack and Accord forms are available.

### 8.4 Faction Setup
For each faction (in any order):
11. Faction receives faction board and places in front of them.
12. ARBITER distributes starting assets per Artifact 08 §7.
13. Faction places starting presence tokens in starting districts.
14. Faction places starting structures in starting districts (if any).
15. Faction places Public Standing marker at starting position on shared track.
16. Faction assembles covert operations deck (see Artifact 08 §8.1).
17. Faction confirms political acts deck.
18. Faction receives Emergency Response card (sealed) and places in designated slot.
19. Faction receives classified directive (sealed) from ARBITER and reads privately.

### 8.5 Final State Verification
20. ARBITER confirms: all five faction presence placements are correct.
21. ARBITER confirms: ARBITER Dominance Marker is at Chorus Node.
22. ARBITER confirms: Portrait board initialized.
23. ARBITER announces: "The Table is convened. Round 1 begins."
